📋 Software Requirements Specification

Online Bookstore - Full Stack Web Application

Project: Online Bookstore
Version: 1.0.0
Technology: PHP 8+, MySQL, Bootstrap 5
Status: Complete

1. Introduction

1.1 Purpose

To build a fully functional online bookstore where users can browse, search, and purchase books, and administrators can manage the entire system.

1.2 Scope

The system provides user registration/authentication, book catalog with search/filter, shopping cart, order processing, admin dashboard with analytics, and full CRUD operations for all modules.

2. Functional Requirements

2.1 Authentication Module

  • FR-01 User shall be able to register with username, email, and password
  • FR-02 User shall be able to login with email and password
  • FR-03 System shall hash passwords using password_hash()
  • FR-04 System shall maintain user sessions
  • FR-05 System shall support forgot/reset password flow
  • FR-06 System shall implement role-based access (Admin/User)

2.2 Admin Module

  • FR-07 Admin shall view dashboard with total users, books, orders, revenue
  • FR-08 Admin shall manage books (Create, Read, Update, Delete)
  • FR-09 Admin shall manage categories (CRUD)
  • FR-10 Admin shall manage users (CRUD, role toggle)
  • FR-11 Admin shall manage orders (CRUD, status update)
  • FR-12 Admin shall search and filter data

2.3 User Module

  • FR-13 User shall browse books with search and filters
  • FR-14 User shall view book details
  • FR-15 User shall add/remove books from cart
  • FR-16 User shall place orders
  • FR-17 User shall view order history
  • FR-18 User shall update profile with picture upload

2.4 Shopping Module

  • FR-19 System shall support search by title/author
  • FR-20 System shall filter by category
  • FR-21 System shall filter by price range
  • FR-22 System shall sort books
  • FR-23 System shall implement pagination
  • FR-24 System shall manage stock inventory

3. Non-Functional Requirements

  • NFR-01 System shall use prepared statements to prevent SQL injection
  • NFR-02 System shall use htmlspecialchars() to prevent XSS
  • NFR-03 System shall validate all inputs server-side
  • NFR-04 System shall be responsive (Bootstrap 5)
  • NFR-05 System shall run on XAMPP without additional configuration
  • NFR-06 System shall have a normalized database with foreign keys
  • NFR-07 System shall use Bootstrap confirmation modals before deletion

4. Database Requirements

TableDescriptionKey Constraints
rolesUser rolesPRIMARY KEY (id)
usersUser accountsPK (id), FK (role_id), UNIQUE (email, username)
categoriesBook categoriesPK (id), UNIQUE (name)
booksBook inventoryPK (id), FK (category_id), INDEXES (title, author, price)
cartShopping cartPK (id), FK (user_id, book_id)
ordersOrder recordsPK (id), FK (user_id), INDEX (status, order_date)
